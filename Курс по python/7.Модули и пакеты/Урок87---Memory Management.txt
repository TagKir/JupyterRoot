Оперативная память разделяется на 2 типа-стек и куча
Стек(как тот,который я писал в 75 уроке)-работает лучше кучи
Имена(ссылки)-лежат в стеке,но связаны с объектами в куче
Ссылки - имя,указывающее на объект в куче
Объекты
Имена - метки объектов
В принципе,один объект может быть "помечен" множеством меток 
Reference count - количество ссылок на один(определённый) объект

Есть две стратегии сборки мусора:
1. Reference counting(только что о нём говорили)
2. Tracing
В чём проблема Reference counting'а:
Подсчёт ссылок не работает на циклических ссылках(у двух объектов ссылающихся друг на друга счётчик reference count'а не становится ниже 1) 
a=b,b=a

Tracing - построение графа объектов,отслеживание поколений объектов
Иногда запускается GC(garbage collector),который маркирует объекты и удаляет "осиротевшие",GC отслеживает длительность жизни объектов и
группирует их по трём поколениям: 0(новые), 1, 2(старые).Когда достигается некий порог потребления памяти в поколении,на нём запускается GC и
на тех,которые младше
Если объект пережил сборку мусора-он переходит в старшее поколение
